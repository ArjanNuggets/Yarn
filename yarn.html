YARN

Qu'est-ce que Yarn

Tout comme NPM, Yarn est un package manager pour JavaScript, c'est à dire qu'il sert à gérer toutes les codépendances des packages que nous utilisons. Il nous permet de prendre des bouts de code créer par d'autre personnes et de les utiliser sur notre site de manière très simple. De plus il permet de mettre à jour nos packages si les developpeurs sortent une nouvelle version de celui ci. Yarn a été créer entre autre par Facebook et Google pour des questions de sécurité et d'efficacité, mais il n'est pas fait pour remplacer NPM car Yarn utilise la bibliothèque de NPM pour ses packages.

Pourquoi Yarn plutôt que NPM

Comme dit plus haut, Yarn et NPM ont le même rôle, donc pourquoi choisir Yarn plutôt que NPM ? Premièrement, Yarn est plus rapide et plus stable que NPM car lorsque que Yarn télécharge un package, il télécharge plusieur partie du package en même temps alors que NPM chaque partie les unes après les autres. De plus, il est plus sécruisé que NPM. Mise à part cela, les deux sont de niveau équivalent.

Les commandes :

yarn init : Permet d'initialiser un nouveau projet, c'est l'équivalent de npm init.

yarn add {nom du package} : permet d'ajouter

on reviendra sur le package.json plus tard

yarn add --> ajoute un package au projet dans le dossier et crée les dépendances

yarn upgrade --> Met à jour les listes de dépendances si on change de version dans le package.json

yarn global add --> Installe un package sur le système d'exploitation de manière général

yarn remove --> Supprime une/des dépédance/s et leur/s références dans le package.json

Vous allez maintenant installer yarn depuis NPM : "npm install yarn --global" dans la console

Verifiez aussi la version de yarn : "yarn --version"

Qu'est-ce que le package.json ?

Le package.json est en quelques sortes le tableau éléctrique du site, soit l'endroit avec tout le cablage et les fusibles (dans ce cas, les différentes versions des packages)
