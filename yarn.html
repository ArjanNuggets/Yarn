YARN

Qu'est-ce que Yarn

Tout comme NPM, Yarn est un package manager pour JavaScript, c'est à dire qu'il sert à gérer toutes les codépendances des packages que nous utilisons. Il nous permet de prendre des bouts de code créer par d'autre personnes et de les utiliser sur notre site de manière très simple. De plus il permet de mettre à jour nos packages si les developpeurs sortent une nouvelle version de celui ci. Yarn a été créer entre autre par Facebook et Google pour des questions de sécurité et d'efficacité, mais il n'est pas fait pour remplacer NPM car Yarn utilise la bibliothèque de NPM pour ses packages mais aussi celle de Bower (une autre bibliothèque).

Pourquoi Yarn plutôt que NPM

Comme dit plus haut, Yarn et NPM ont le même rôle, donc pourquoi choisir Yarn plutôt que NPM ? Premièrement, Yarn est plus rapide et plus stable que NPM car lorsque que Yarn télécharge des packages, il télécharge plusieurs package en même temps alors que NPM chaque package les uns après les autres. De plus, il est plus sécruisé que NPM. Pour finir, lorsque NPM installe les packages, il affiche touts les détails de l'installation ce qui rend l'affichage très peu comprehensible alors que Yarn a un affichage beaucoup plus propre. Mise à part cela, les deux sont de niveau équivalent.

Comment installer Yarn

Il y a plusieurs manière de le faire. Si on à déjà NPM sur l'ordinateur on peut simplement faire "npm install --global yarn". Si on a pas NPM et qu'on est sur mac on fait "brew install yarn". Sur windows, on télécharge l'installer et on le lance.

Les commandes :

• yarn init : Permet d'initialiser un nouveau projet, c'est l'équivalent de npm init.

• yarn -v : Donne la version de Yarn installée.

• yarn add {nom du package} : permet d'installer un package dans notre projet (et dans le package.json)

• yarn remove {nom du package} : permet de supprimer le package de notre projet (supprime aussi du package.json)

• yarn why {nom du package} : explique pourquoi le package est installé et quels package utilisé ce package.

• yarn upgrade {nom du package} : permet de mettre a jour les packages installés (À noter que le nom du package est faculatif si il n'est pas mis Yarn mettra à jour tout les packages.)

• yarn install : sert à installer tout les packages du fichier package.json (crée le Yarn.lock)

Package.json et Yarn.lock

Si tout s'est passé comme prévu normalement vous devez avoir deux fichier. Le package.json et un deuxième le Yarn.lock.

Le fichier Package.json est une liste de tout les packages que le projet à besoin pour foncitonner. Vous pouvez indiquer plusieurs versions des packages. Il sert aussi à lister toutes les infos de votre projet. Son nom, sa version etc...

Le fichier Yarn.lock est une liste qui contient toutes les dépendances que nous avons besoin pour notre projet. Il sert à faire en sorte que tout les membres d'une équipe aient les mêmes versions des packages utilisés. Cela permet d'éviter les bugs dû à des conflits de version. Lorsqu'on lance "yarn install", Yarn va chercher dans ce fichier quelles versions des packages il doit installer.
